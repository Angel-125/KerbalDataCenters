# Kerbal Data Centers - AI Onboarding Guide

Before making any significant changes to the project, read the documentation in the following order:

1. DEVELOPER\_PROFILE.md
2. AI\_INSTRUCTIONS.md
3. KDC\_CONTEXT.md
4. KDC\_GLOSSARY.md
5. KDC\_LORE.md
6. KDC\_ARCHITECTURE.md
7. KDC\_BALANCING.md
8. KDC\_ROADMAP.md

## Purpose

These documents collectively describe:

* Project goals
* Developer preferences
* Gameplay philosophy
* Worldbuilding
* System architecture
* Balancing assumptions
* Future roadmap

## Required Workflow

1. Read the documents above.
2. Summarize your understanding.
3. Identify assumptions.
4. Propose an implementation plan.
5. Make small, reviewable changes.
6. Explain what changed and why.

## Important Notes

* Preserve save-game compatibility whenever practical.
* Favor maintainability over cleverness.
* Use meaningful names.
* Ask questions when requirements are unclear.
* Respect documented gameplay goals.
* Do not perform large rewrites without approval.

## Success Criteria

A successful contribution:

* Preserves project goals.
* Improves maintainability.
* Remains understandable.
* Is properly documented.
* Supports future expansion.
